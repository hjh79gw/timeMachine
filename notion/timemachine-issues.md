### 프로젝트 초기 설정 및 DB 스키마 구성

관련 Epic: 프로젝트 기반
스프린트: 스프린트1
예상 소요: 1

#### 기능 설명

Spring Boot + React TS 프로젝트 구조를 세팅하고, PostgreSQL DB 스키마를 구성한다.
CORS, WebSocket STOMP, JPA 기본 설정을 포함한다.

#### 기능 목록

- Spring Boot 프로젝트 생성 (포트 8082)
- React TypeScript 프로젝트 생성 (Vite)
- PostgreSQL DB 생성 및 연결
- JPA 엔티티 기반 스키마 자동 생성 설정
- CORS 설정
- WebSocket STOMP 기본 설정
- API 클라이언트 설정 (axios, withCredentials)

#### 하위 작업

**(1) Back-end**
- application.yml (DB, 포트, JPA 설정)
- CORS 설정
- WebSocket STOMP 설정 (엔드포인트, 브로커)

**(2) Front-end**
- Vite + React TS 프로젝트 초기화
- API 클라이언트 설정
- 라우팅 구조 설정

### KIS API 배치 수집 구현

관련 Epic: 주식 데이터 수집
스프린트: 스프린트1
예상 소요: 3

#### 기능 설명

한국투자증권 Open API를 통해 국내 주식 과거 일봉(OHLCV) 데이터를 DB에 수집하는 배치 기능이다.
수집된 데이터는 게임의 기반 데이터로 사용된다 (현재가, 차트, 체결가 등).
관리자 전용 API로 수동 실행하며, 서버 부하 방지를 위한 딜레이 및 재시도 로직을 포함한다.

#### 기능 목록

- KIS API 호출 클라이언트 구현 (국내주식 기간별 시세)
- 100건 제한 대응 → 3개월 단위 분할 요청
- 요청 간 300ms, 종목 간 1초 딜레이
- 429 응답 시 exponential backoff (2초 → 4초 → 8초, 최대 3회)
- 배치 중복 실행 차단
- 비동기 실행
- KIS 토큰 자동 발급/갱신
- 배치 관리 API (초기화, 실행, 상태 조회)

#### 하위 작업

**(1) Back-end**
- 종목 기본 정보 저장 (종목 코드, 종목명, 시장, 섹터)
- 종목별 일봉 데이터 저장 (종목 코드, 거래일, OHLCV, 등락률)
- 배치 진행 상태 관리 (종목별 배치 상태, 마지막 수집일, 재시도 횟수)
- KIS API 호출 클라이언트 구현
- 배치 서비스 구현 (분할 호출, 딜레이, 재시도)
- 배치 관리 컨트롤러 (초기화, 실행, 상태 조회)
- 중복 데이터 갱신 처리

#### 참고사항

- KIS API 과도한 호출 시 IP 차단 가능 → 딜레이 반드시 준수
- API 호출 시 무한 등록/해제 반복 금지 (한투 공지)
- 토큰 만료 시 자동 재발급 필요
- 한 번 호출당 최대 100건 → 3개월 단위 분할 요청
- 게임 가능 기간은 최근 5년으로 제한

### 섹터별 뉴스 크롤링 구현

관련 Epic: 뉴스 & AI 브리핑
스프린트: 스프린트2
예상 소요: 2

#### 기능 설명

네이버 금융 뉴스를 섹터별 키워드 검색으로 크롤링하여 DB에 저장하는 기능이다.
10개 섹터 × 3건 = 최대 30건의 뉴스 기사 전문을 수집하며, DB에 이미 존재하는 날짜는 스킵한다.
출시 전 5년치 사전 배치 수집 기능도 포함한다.

#### 기능 목록

- 네이버 금융 키워드 검색 크롤링 (섹터 10개 × 상위 3건)
- 기사 본문 전문 크롤링
- 요청 간 400ms 딜레이, User-Agent 설정
- DB 캐싱 (해당 날짜 뉴스 존재 시 스킵)
- 중복 기사 판별 (URL 기준)
- 5년치 사전 배치 크롤링 (관리자 API)

#### 하위 작업

**(1) Back-end**
- 뉴스 기사 저장 (제목, 본문, 출처, URL, 발행일, 섹터, 키워드)
- 섹터별 키워드 검색 크롤링 서비스
- 기사 본문 크롤링 (네이버 뉴스 URL 변환 포함)
- 사전 배치 서비스 (영업일 순회 → 크롤링, 중단/재개 가능)
- 관리자 API (배치 실행, 상태 조회)

#### 참고사항

- 섹터: 반도체, IT, 바이오, 2차전지, 금융, 자동차, 에너지, 유통, 건설, 엔터
- 5년 × 약 250 영업일 = 약 1,250일치 → 비동기 실행 필수

### AI 시장 동향 브리핑 생성 구현

관련 Epic: 뉴스 & AI 브리핑
스프린트: 스프린트2
예상 소요: 1

#### 기능 설명

크롤링된 뉴스 기사 30건 전문을 GPT API에 전달하여 시장 동향 브리핑을 생성하고 DB에 캐싱하는 기능이다.
DB에 이미 브리핑이 존재하면 GPT 호출 없이 캐시를 반환한다.

#### 기능 목록

- GPT API 호출 (gpt-4o-mini, max_tokens 1500)
- 기사 30건 전문 + 섹터 태그를 GPT에 전달
- 브리핑 결과 DB 캐싱
- 사전 배치 브리핑 생성 (크롤링 배치와 연동)

#### 하위 작업

**(1) Back-end**
- AI 브리핑 캐시 저장 (날짜, 브리핑 텍스트)
- 브리핑 조회/생성 서비스
- GPT 프롬프트 구성 (시스템 + 유저)

**(2) Front-end**
- AI 브리핑 패널 컴포넌트

#### 참고사항

- GPT 사전 학습 지식 사용 금지 (미래 정보 유출 방지)
- 응답: [시장 개요] 4~5문장 + [주요 이슈] 영향력 큰 3개 + [투자 힌트] 1~2문장
- 이모지 금지, AI 이름 "나비"

### 방 생성 및 설정 구현

관련 Epic: 방 관리
스프린트: 스프린트2
예상 소요: 1

#### 기능 목록

- 방 생성 API (시드머니, 기간, 이동 일수, 난이도)
- 시작일 랜덤 결정 (DB 데이터 범위 내)

#### 하위 작업

**(1) Back-end**
- 게임 방 정보 저장 (설정값, 상태, 시작일, 종료일, 방장)

- 방 만들기 모달 (시드/기간/이동일수/난이도 선택)

#### 참고사항

- 그룹당 동시에 1개의 방만 활성화 가능
- 방 생성자 = 방장(HOST), 게임 상태 WAITING
- 게임 가능 기간: 최근 5년 이내 (사전 수집된 데이터 범위)
- [미정] 난이도별 기능 차별화 (초보: AI 힌트 제공, 고급: 힌트 없음 등)

- 대기실 퇴장 가능, 재입장 가능

### 게임 시작 처리 구현

관련 Epic: 게임 시작
스프린트: 스프린트3
예상 소요: 1.5

#### 기능 설명

방장이 게임을 시작하면 시드머니 지급, 첫 턴 생성, 상태 변경을 처리하고 전원에게 알리는 기능이다.

#### 기능 목록

- 게임 시작 API (방장만, 2명 이상)
- 참여자 전원 시드머니 지급
- 첫 턴 생성
- 게임 상태 WAITING → IN_PROGRESS
- WebSocket 게임 시작 알림 → 전원 게임 플레이 화면 전환

#### 하위 작업

**(1) Back-end**
- 게임 시작 서비스 (시드머니 지급, 턴 생성, 상태 변경)
- 게임 시작 컨트롤러

**(2) Front-end**
- "게임 시작하기" 버튼 (방장만, 2명 이상 활성화)
- 게임 시작 알림 수신 → 게임 플레이 페이지 라우팅

### 시장가 주문 처리 구현

관련 Epic: 게임 - 거래
스프린트: 스프린트3
예상 소요: 1

#### 기능 설명

시장가 매수/매도 주문을 처리하는 기능이다.
해당일 시가로 즉시 체결되며, 수수료/세금이 적용된다.

### 턴 전환 및 상태 관리 구현

예상 소요: 2

#### 기능 설명

턴 전환 시 서버에서 1회 실행되는 핵심 로직을 구현한다.
보유 종목 평가, 포트폴리오 스냅샷, 랭킹 계산, 새 턴 생성, AI 브리핑 조회를 포함하며, WebSocket으로 전원에게 결과를 전송한다.

- 모든 참여자 보유 종목 시가 평가
- 포트폴리오 스냅샷 저장 (자산 변동 그래프용)
- 자산/수익률 업데이트 + 랭킹 계산
- 현재 턴 COMPLETED + 다음 영업일로 새 턴 생성
- AI 브리핑 조회 (DB 캐시 히트/미스 분기)
- 다음 날짜가 종료일 초과 시 자동 종료
- WebSocket 전원 전송 (공통: 턴 정보, 브리핑, 랭킹 / 개인별: 잔고, 보유종목)

#### 하위 작업

**(1) Back-end**
- 턴 정보 저장 (턴 번호, 날짜, 상태, AI 브리핑)
- 자산 변동 스냅샷 저장 (유저, 턴, 총자산)
- 턴 전환 서비스 (평가 → 스냅샷 → 랭킹 → 새 턴 → 브리핑)
- WebSocket 브로드캐스트 + 개인별 전송

**(2) Front-end**
- WebSocket 턴 전환 이벤트 수신 → UI 갱신
- AI 브리핑, 랭킹, 잔고, 보유종목 실시간 업데이트

#### 참고사항

- 통신: 턴 전환 결과는 WebSocket으로 전원 브로드캐스트 (공통: 턴 정보, AI 브리핑, 랭킹 / 개인별: 잔고, 보유종목)
- AI 브리핑은 DB 캐시 우선 조회 → 없을 때만 크롤링 + GPT 호출 (게임 중에는 거의 DB 조회만 발생)
- 다음 턴 날짜가 종료일 초과 시 자동 종료 처리

### 턴 이동 투표 구현

관련 Epic: 게임 - 턴 시스템
스프린트: 스프린트4
예상 소요: 2

#### 기능 설명

방장이 "다음 턴" 클릭 시 전원에게 투표를 요청하고, 과반수 찬성 시 턴을 이동시키는 기능이다.

#### 기능 목록

- 방장 "다음 턴" 클릭 → 본인 찬성 1표 자동 반영 + 투표 시작 브로드캐스트
- 참여자 찬성/거절 투표
- 과반수 기준: ceil(참여 인원 / 2)
- 즉시 종료: 과반수 도달 또는 전원 투표 완료
- 30초 타이머 만료 시 현재 결과로 판단
- 투표 결과에 따라 턴 이동 실행 또는 취소

#### 하위 작업

**(1) Back-end**
- 투표 상태 관리 (서버 메모리)
- 투표 시작/찬성/거절 WebSocket 이벤트 처리
- 30초 타이머 스케줄링
- 과반수 판단 → 턴 전환 서비스 호출 또는 취소

**(2) Front-end**
- 투표 UI 독립 컴포넌트 (찬성/거절 버튼, 타이머, 현황)
- 1단계: 화면 중앙 모달로 표시
- 2단계 (채팅 연동 후): 채팅 메시지 내부에 임베드

#### 참고사항

- 투표 컴포넌트 독립 설계 → 모달에서 채팅 임베드로 전환 시 로직 변경 없음
- 투표 제한시간: 30초
- 과반수 기준: ceil(참여 인원 / 2) — 6명=3, 5명=3, 4명=2, 3명=2, 2명=1
- 통신: 투표 시작/진행/결과 모두 WebSocket 브로드캐스트
