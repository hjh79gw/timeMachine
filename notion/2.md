# 뉴스 수집 및 저장 파이프라인 구현

관련 Epic: 뉴스 데이터 파이프라인
스프린트: 스프린트1
예상 소요: 2

## 기능 설명

외부 금융 뉴스 데이터를 주기적으로 수집하고, 서비스에서 사용할 수 있도록 정제 및 저장하는 기능이다.

뉴스 제목, 본문, 썸네일, 출처, 발행 시각, 원문 링크 등의 데이터를 저장하며, 중복 기사 제거와 기본 텍스트 정제를 포함한다.

이 기능은 이후 뉴스 조회, 태깅, RAG 임베딩 생성, AI 브리핑 생성의 기반이 되는 데이터 파이프라인 역할을 한다.

---

## 기능 목록

- 외부 뉴스 API 또는 크롤링을 통한 뉴스 수집
- 주기적 수집을 위한 배치 실행
- 뉴스 원문 데이터 파싱
- 뉴스 본문 HTML 제거 및 텍스트 정제
- 중복 기사 판별 및 제거
- 뉴스 DB 저장
- 저장 성공/실패 로깅

---

## 하위 작업

### (1) Back-end

- News 엔티티 및 Repository 구현
- 뉴스 수집용 Client 구현
- 외부 응답값을 내부 DTO로 변환하는 로직 구현
- 뉴스 본문 정제 로직 구현
- 중복 기사 판별 로직 구현
- 뉴스 저장 Service 구현
- 배치 실행용 Scheduler 구현
- 수집 결과 로깅 및 예외 처리 구현

---

### (2) Front-end

- 없음

---

### (3) ETC

- 수집 대상 뉴스 소스 선정
- 수집 주기 결정
- 중복 판별 기준 정의
    - 예: 동일 제목 + 동일 언론사 + 발행 시각 유사

## 참고사항

- 본 이슈는 뉴스 서비스 전체 기능의 선행 작업이다.
- 태깅, 임베딩 생성, 벡터 DB 저장, 뉴스 조회 API 구현 전에 완료되어야 한다.
- MVP 단계에서는 안정적인 수집 및 저장이 우선이며, 고급 필터링이나 AI 처리는 포함하지 않는다.
- 수집 실패 시 전체 배치가 중단되지 않도록 예외 처리가 필요하다.
